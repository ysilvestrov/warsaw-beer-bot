# Браузерне розширення «Warsaw Beer Overlay» — встановлення та налаштування

Розширення накладає на сторінки крафтових магазинів твій особистий статус
**«вже пив»** і **твою оцінку** для кожного пива, яке збігається з твоєю історією
Untappd. Працює на:

- `beerrepublic.eu` (і піддомени, напр. `www.beerrepublic.eu`)
- `onemorebeer.pl` (і піддомени)

Щоб усе запрацювало, потрібні три речі: (1) завантажити свою історію пив у бота,
(2) отримати токен доступу, (3) встановити й налаштувати розширення. Нижче — по кроках.

---

## Передумови

- Браузер на базі Chromium: **Chrome, Edge, Brave або Opera** (розширення формату MV3).
- Акаунт **Untappd** з історією чекінів (бажано Supporter — щоб експортувати історію).
- _Node.js потрібен лише розробникам, що збирають із сорсів (див. примітку в Частині 2)._

---

## Частина 1. Реєстрація в боті та завантаження списку пив

Це робиться **один раз** у Telegram-боті. Саме історія Untappd дає розширенню дані
про те, що ти вже пив (без неї бейджі не з'являться).

### 1.1. Запустити бота й обрати мову

1. Відкрий бота в Telegram, надішли **`/start`**.
2. За потреби зміни мову: **`/lang`**.

### 1.2. Прив'язати акаунт Untappd

Надішли:

```
/link <username>
```

де `<username>` — твій нік в Untappd (можна повний URL `untappd.com/user/<username>`).
У відповідь: `✅ Прив'язано до untappd.com/user/<username>`.

### 1.3. Завантажити історію пив (імпорт) — це і є «список пив»

1. Надішли команду **`/import`**.
2. Бот попросить файл експорту з Untappd: **CSV, JSON або ZIP (до 20 MB)**.
3. Де взяти експорт: в Untappd → **Supporter → Account → Download History**.
   - Якщо JSON великий (> 20 MB) — **запакуй його в ZIP** (стискається приблизно в 10×)
     і надішли ZIP: Telegram не дасть боту скачати файл більший за 20 MB.
4. Надішли файл боту як **документ** (вкладення, не текст).
5. Бот покаже прогрес і завершить: `✅ Імпортовано N чекінів`.

Після цього бот знає, що саме ти пив і як оцінив — це й використає розширення.

> Можна оновлювати історію будь-коли, надіславши новий експорт через `/import`.

### 1.4. Отримати токен доступу для розширення

Надішли:

```
/extension
```

Бот відповість інструкцією та токеном у вигляді копійованого блоку
(`<code>…</code>`), а також **адресою API**.

> **Важливо:** токен ротується **1:1** — кожен виклик `/extension` створює новий токен
> і **анулює попередній**. Якщо колись зробиш `/extension` ще раз, не забудь оновити
> токен у налаштуваннях розширення.

Скопіюй токен — він знадобиться в Частині 3.

## Частина 2. Встановлення розширення

Розширення видає сам бот — збирати з сорсів **не потрібно**.

### 2.1. Отримати zip

Надішли **`/extension`**. Бот пришле токен (Частина 1.4) **і файл**
`warsaw-beer-overlay-<версія>.zip`. Збережи його й **розпакуй у постійну теку**
(напр. `~/warsaw-beer-overlay/`) — саме її ти завантажуватимеш у браузер і саме
поверх неї оновлюватимеш.

### 2.2. Завантажити в браузер (на прикладі Chrome)

1. Відкрий **`chrome://extensions`** (в Edge — `edge://extensions`, Brave — `brave://extensions`).
2. Увімкни **«Режим розробника» / «Developer mode»** (перемикач угорі праворуч).
3. Натисни **«Завантажити розпаковане» / «Load unpacked»**.
4. Обери теку, у яку розпакував zip.

Розширення з'явиться у списку. Закріпи його іконку на панелі (необов'язково, але зручно).

### 2.3. Оновлення

Коли вийде нова версія, бот сам пришле новий zip. **Розпакуй його поверх тієї ж
теки** (із заміною) і натисни **↻** на картці розширення в `chrome://extensions`.
Токен і налаштування збережуться — ID розширення зафіксовано.

> **Для розробників: збірка з сорсів.** Якщо хочеш зібрати сам:
> `cd extension && npm install && npm run build` → завантаж `extension/dist/`.
> `npm run package` додатково спакує `dist/` у zip.

## Частина 3. Налаштування розширення

1. Відкрий сторінку налаштувань розширення:
   - правий клік по іконці розширення → **«Параметри» / «Options»**, або
   - `chrome://extensions` → у картці розширення → **«Деталі» → «Параметри розширення»**.
2. У поле **«Token»** встав токен, отриманий від бота (`/extension`).
3. Поле **«API URL»** залиш за замовчуванням:
   `https://beer-api.ysilvestrov-ai.uk`
   (зміни лише якщо в тебе власний інстанс API).
4. Натисни **«Save»**.
   - Якщо ти змінив URL на нестандартний, браузер запитає дозвіл на доступ до цього
     хоста — погодься, інакше розширення не зможе звертатися до API.
5. Натисни **«Test connection»**:
   - **✅ Connected** — усе гаразд.
   - **❌ Failed (health)** — недоступний API/URL (перевір адресу та інтернет).
   - **❌ Failed (unauthorized)** — невірний або застарілий токен (зроби `/extension` ще раз).
   - **❌ Failed (network)** — браузер не зміг звернутися до API: немає дозволу на хост
     (для нестандартного URL) або немає мережі.
   - **❌ Failed (server)** — тимчасова помилка сервера, спробуй пізніше.

---

## Частина 4. Використання

1. Зайди на підтримуваний магазин (`beerrepublic.eu` або `onemorebeer.pl`).
2. Розширення **автоматично** зчитує сітку пив на сторінці, надсилає її боту на звірку
   й малює **кутовий бейдж** на тих позиціях, які ти **вже пив**, разом із **твоєю оцінкою**.
3. Це працює і для SPA-навігації: коли магазин перемальовує список (фільтри, пагінація),
   накладка оновлюється сама.

> Бейджі з'являються **тільки** для пив, що збіглися з твоєю історією. Якщо ти ще не
> робив `/import`, бейджів не буде — спершу заверши Частину 1.

## Усунення несправностей

| Симптом | Що зробити |
|---|---|
| `Test connection` → **health** | Перевір поле «API URL» і доступ до інтернету/тунелю. |
| `Test connection` → **unauthorized** | Токен застарів або невірний. Зроби `/extension`, встав новий токен, **Save**. |
| `Test connection` → **network** | Браузер не дотягнувся до API: для нестандартного URL дай дозвіл на хост (**Save** ще раз або `chrome://extensions → Деталі`); перевір мережу. |
| Бейджі не з'являються | 1) Ти на підтримуваному сайті? 2) Чи зроблено `/import`? 3) Перезавантаж сторінку. |
| Зробив `/extension` повторно — перестало працювати | Старий токен анульовано. Встав **новий** токен у налаштуваннях і **Save**. |
| Змінив API URL — не фетчить | Під час **Save** браузер просить дозвіл на хост — погодься (або додай дозвіл у `chrome://extensions → Деталі`). |

## Коротко (швидкий старт)

1. У боті: `/start` → `/link <username>` → `/import` (надіслати експорт Untappd) → `/extension` (скопіювати токен **і** зберегти zip).
2. Розпакуй zip у постійну теку (напр. `~/warsaw-beer-overlay/`).
3. `chrome://extensions` → Developer mode → Load unpacked → ця тека.
4. Options розширення → встав токен → Save → Test connection → ✅.
5. Зайди на `beerrepublic.eu` / `onemorebeer.pl` — бейджі «вже пив» + оцінка з'являться самі.
6. Оновлення: бот пришле новий zip → розпакуй поверх тієї ж теки → **↻ reload**.
