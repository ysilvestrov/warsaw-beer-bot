# Warsaw Beer Crawler Bot — гайд по командах

Бот допомагає знайти **цікаве непите пиво** у варшавських пабах і
зібрати **пішохідний маршрут** через них.

Дані бере з двох джерел:
- **ontap.pl/warszawa** — що зараз налите на крані;
- **твій Untappd-профіль** — що ти вже пив.

«Цікаве непите» = є на крані прямо зараз, не у твоїх чекінах,
проходить твої фільтри стилю / рейтингу.

---

## Швидкий старт

1. `/start` — реєстрація.
2. `/link <твій-untappd-username>` — щоб бот знав, що ти вже пив.
3. (опційно) `/import` — заллє повну історію з Untappd-експорту, краще ніж
   тільки публічна сторінка (там видно лише останні ~25 чекінів).
4. `/refresh` — підтягнути свіжі крани й твою історію *прямо зараз*.
5. `/newbeers` — топ непитих пив на поточних кранах.
6. `/route 5` — маршрут, що покриває щонайменше 5 непитих пив.

---

## Команди

### `/start`
Створює твій профіль (ключ — Telegram ID) і друкує коротку інструкцію.
Виконати один раз. Безпечно повторювати — нічого не зламає, профіль не
дублюється.

### `/link <username>`
Прив'язує твій публічний Untappd-профіль. Приймає:
- bare-юзернейм: `/link yuriy`
- повний URL: `/link https://untappd.com/user/yuriy`
- з www: `/link https://www.untappd.com/user/yuriy`

**Важливо:** профіль на untappd.com має бути **публічним**, бо бот ходить
без cookie. Якщо приватний — скрейпер бачитиме 0 чекінів.

Відповідь: `✅ Прив'язано до untappd.com/user/<username>`.

### `/import`
Приймає експорт з Untappd для повного бекфілу історії (на відміну від
скрейпера, який бачить лише останні ~25 чекінів).

**Як отримати експорт:** Untappd → Account → Settings → Supporter →
Download History. Завантажиш `.csv` або `.json`. Архіви `.zip` теж приймаються.

**Обмеження:**
- Telegram Bot API не дає скачувати файли > **20 MB**. Великий JSON
  легко перевалює — запакуй у ZIP (стискається ≈10×) і надішли його.
- Будь-який інший формат (`.txt`, `.xlsx`, …) не приймемо.

Бот стрімить файл і вантажить чекіни батчами по 500. Поки йде імпорт,
ти бачиш живий лічильник: `⏳ Імпортовано N…`. У кінці —
`✅ Імпортовано N чекінів (CSV/JSON/ZIP).`. Імпорт ідемпотентний:
повторне завантаження тих же чекінів нічого не задвоїть.

### `/newbeers`
Топ-15 цікавих непитих пив у місті, **згруповано по пиву**.

Приклад відповіді (HTML, жирні назви):

```
1. Browar Stu Mostów Salamander  ⭐ 3.92  ·  6,5%
     · Cuda na kiju, PiwPaw, Same Krafty +2 інших
2. Pinta Atak Chmielu  ⭐ 3.81  ·  6,1%
     · Hoppy & Brewy
…
```

- Якщо те ж пиво на кранах кількох пабів — це **один рядок**, паби йдуть
  списком. Якщо пабів більше трьох, лишаємо перші три та `+N інших`.
- Сортування: рейтинг (з Untappd) ↓ → кількість пабів ↓ → назва ↑.
- ABV — окремий чіп після рейтингу, якщо ontap дав цю інформацію.
- Беремо тільки ті крани, що пройшли матчинг проти Untappd-каталогу
  (інакше бот не знає, чи ти це пив). Невідомі пива на кранах поки не
  показуємо.
- **Vintages**: якщо в Untappd твоя історія має кілька років того ж пива
  ("Buzdygan Rozkoszy 2024", "2025"), а ontap показує просто
  "Buzdygan Rozkoszy" — бот дивиться на ABV. Збігається з найсвіжішим
  vintage → зараховуємо як «пите»; не збігається → шукаємо серед старіших;
  жодне не співпало → беремо найсвіжіший entry за замовчуванням. Тобто нова
  партія з тим же ABV, що й твої попередні чекіни, теж буде «пита».
- Якщо нічого цікавого — `Нічого цікавого — спробуй /refresh.`

Фільтри з `/filters` (стиль, мінімальний рейтинг) застосовуються до
кандидатів *перед* групуванням.

### `/route [N]`
Будує **пішохідний маршрут**, що покриває щонайменше `N` непитих пив,
з мінімальною сумарною дистанцією.

- Без аргументу `N` береться з `default_route_n` твоїх фільтрів,
  інакше — з `DEFAULT_ROUTE_N` у `.env` (5 за замовчуванням).
- Перший виклик трохи довший (хвилину) — будується N×N матриця пішохідних
  відстаней через OSRM; якщо OSRM відмовив — fallback на haversine
  (геометричну відстань).
- Кінцевих/стартових точок не фіксуємо — повертаємо оптимальний open-TSP.
- Жорстких лімітів на дистанцію/кількість пабів немає; у хедері відповіді
  завжди є рядок типу `Маршрут: ≥5 нових пив, покрито 7, ≈ 3.4 км, 5 пабів`,
  тож масштаб видно одразу.

### `/filters`
Інлайн-клавіатура для зміни фільтрів цікавих пив:
- стилі: `IPA`, `Pils`, `Stout`, `Sour` (можна кілька — toggle);
- `min 3.5` / `min 3.8` — мінімальний Untappd-рейтинг;
- `Скинути` — скидає всі фільтри.

Поточні фільтри показуються одразу в тілі повідомлення.

### `/refresh`
Примусово запускає обидва пайплайни прямо зараз:
1. Обхід ontap.pl/warszawa: ~50 пабів послідовно (~3 хв).
2. Скрейп твого untappd-профілю.

**Кулдаун:** 5 хв на користувача — щоб не довбати джерела.

Поки оновлення йде, бот редагує одне й те ж повідомлення (раз на ~2c):

```
⏳ Оновлюю…
🍻 ontap: парсю індекс…
🍻 ontap: 12/47 — bracka
🍻 ontap: 47/47 пабів
👤 untappd: 1/3 — yuriy
✅ Готово.
```

Базове оновлення працює і **без** `/refresh`: ontap-обхід — кожні 12 годин,
untappd — щодоби о 03:00 UTC.

## Поради й часті питання

**Що робити, якщо `/import` падає на великому JSON?**
Запакуй у ZIP перед надсиланням. ZIP розпаковується на льоту, і файл
влазить у 20-MB ліміт Telegram.

**Чому `/newbeers` нічого не показує?**
- Можливо, ще не запустив `/refresh` — у БД нема свіжого snapshot.
- Можливо, ввімкнуті жорсткі фільтри (`/filters` → `Скинути`).
- Можливо, на ontap.pl справді нічого нового конкретно для тебе.

**Чому в `/newbeers` нема пива, що бачу на ontap.pl?**
Бот показує тільки матчені крани (де ми впевнені, що це за пиво в каталозі).
Якщо пиво нове й парсер його ще не зматчив — у списку не з'явиться.

**Як змінити частоту cron?**
Це конфіг у коді (`src/index.ts`, виклики `cron.schedule`). Для
маневреності у проді — реденьку. Якщо треба змінити — окремий PR.

**Куди йти, коли щось не так?**
Логи на хості: `sudo journalctl -u warsaw-beer-bot -f` — структуровані
JSON-рядки `pino`. `bot error` зазвичай означає або зламаний токен,
або крайній випадок у парсері.
