# CWS distribution migration — design (#247)

**Issue:** GH #247 (серія `chrome-web-store`)
**Дата:** 2026-07-10
**Статус:** design → docs-only implementation

## Контекст

Публікація в Chrome Web Store змінює канал дистрибуції розширення. Наслідки:

1. **Зміна extension ID.** Store-збірка без `key` → CWS присвоює свій ID
   (`fdelmnhijeiojadcaihfdpecfcldbndg`), відмінний від `key`-pinned ID unpacked-збірки.
   `chrome.storage.local` прив'язаний до ID → у store-версії токен/налаштування порожні.
2. **Off-store канал (bot broadcast + `extension_releases` + hash-match zip) стає legacy** —
   CWS оновлює користувачів сам.

## Рішення (dialog з користувачем, 2026-07-10)

1. **Модель дистрибуції — перехідний період, потім повний перехід (варіант #3).** CWS —
   основний канал; off-store bot-канал працює паралельно на час переходу (рев'ю CWS
   безстрокове), потім ретайриться. Unpacked-збірка з `key` лишається як dev-режим мейнтейнера.
2. **Аплоад у CWS — ручний** через dashboard. Автоматизація (`chrome-webstore-upload-cli` +
   OAuth) відкладена в окремий issue **#266** (рідкі релізи не виправдовують налаштування секретів).
3. **Off-store канал — ретайр повністю** на cutover (не text-анонси). Cutover-дії (broadcast
   міграції → ретайр `extension_releases`/broadcast) — окремий операційний issue **#267**.

## Скоуп цього PR — **лише документація**

Жодних змін коду. Пайплайн-зміни, ретайр broadcast і фінальний «store-primary» флip доків —
у #267; автоматизація аплоаду — у #266.

- **`spec.md` §6.4 (нова)** — запис моделі: CWS основний + auto-update; store ID; наслідок
  зміни ID; реліз під час переходу = обидва канали; dev = unpacked+key; ретайр на cutover
  (#267); аплоад ручний (#266). Плюс **чернетка міграційного повідомлення** тестерам.
- **`spec.md` §6.1** — позначено як legacy на час переходу (вказівник на §6.4).
- **`docs/extension-install-uk.md`** — нотатка на початку «Частини 2»: розширення подано в CWS;
  після схвалення рекомендований спосіб — зі стору (auto-update, інший ID → заново вставити
  токен); поки триває рев'ю — unpacked через бота (як зараз). Кроки unpacked **не** видаляємо —
  це робота cutover (#267).
