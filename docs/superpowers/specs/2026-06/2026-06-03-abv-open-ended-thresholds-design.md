# ABV filter: open-ended thresholds — design

> **Дата:** 2026-06-03
> **Статус:** approved (brainstorming) → готується план
> **Спека-істина:** оновлює `spec.md` §3.9, §4 (`/filters`)
> **Суперсідить:** ABV-бакети (closed bands) з `2026-06-03-filters-surface-design.md`.

## 1. Проблема

ABV-фільтр — це **закриті суміжні смуги** (`≤5%`, `5–7%`, `7–9%`, `9%+`) із
**single-select**. Найчастіша задача — «тільки міцне» або «тільки легке» —
не виконується:

- «міцне» = `≥7%`, тобто `7–9%` **∪** `9%+`; але single-select дозволяє лише
  одну смугу, а `7–9%` **відсікає** монстрів `9%+`;
- «легке» працює лише випадково, бо `≤5%` відкрита знизу.

**Корінь:** потрібні **відкриті пороги** (`≤X` / `X+`), а не партиція діапазону.

## 2. Рішення (узгоджено)

Замінити закриті смуги на **відкриті порогові пресети**, single-select,
tap-active-to-clear (як зараз). `≤` — включно; `+` = «і вище», включно.

| chip | `abv_min` | `abv_max` |
|------|-----------|-----------|
| `≤3.5%` | null | 3.5 |
| `≤5%`   | null | 5 |
| `5%+`   | 5 | null |
| `7%+`   | 7 | null |
| `9%+`   | 9 | null |

- `≤5%` і `9%+` мають **ті самі `(min,max)`**, що старі смуги `≤5%`/`9%+`, тож
  попередні вибори лишаються чинними; стають stale лише старі `5–7%`/`7–9%`.
- `7%+` тепер коректно включає й `9%+`-монстрів (немає верхньої межі).

**Розкладка — два ряди** (кепи / флори):
```
[ ≤3.5% ][ ≤5% ]
[ 5%+ ][ 7%+ ][ 9%+ ]
```

## 3. Чесна обробка stale-діапазонів

Користувач, що раніше тапнув `5–7%`/`7–9%`, має збережені
`abv_min`/`abv_max`, яким **жоден новий чип не відповідає**. Зараз зведення
бере мітку з ключа чипа → показало б «будь-яка», поки фільтр **мовчки
застосовується** — пастка.

**Фікс:** рядок-зведення рахує ABV з **сирих** `(abv_min, abv_max)` через нову
чисту `formatAbvRange(min, max)`:

```
formatAbvRange(min, max): string | null
  // (null,  null) → null            (виклик підставляє t('filters.any'))
  // (null,  m)    → `≤${m}%`
  // (n,     null) → `${n}%+`
  // (n,     m)    → `${n}–${m}%`     (stale bounded; видно й чесно)
```

Тож stale-діапазон **видно** у зведенні (напр. «Міцність: 5–7%») і чиститься
через ♻️, навіть якщо жоден чип не має ✓. `bucketForRange` лишається — але
**лише** щоб вирішити, який чип показує ✓ (точний збіг `(min,max)` з пресетом).

## 4. Компоненти / точки дотику

- **`src/domain/filters.ts`:**
  - Перейменувати `ABV_BUCKETS` → `ABV_PRESETS` (семантика змінилась: пороги, не
    смуги). Тип `AbvBucket` → `AbvPreset`.
  - Нові 5 пресетів (див. §2). Ключі ascii-safe: `lte3_5`, `lte5`, `gte5`,
    `gte7`, `gte9`.
  - `bucketForRange(abvMin, abvMax)` — без змін у сигнатурі, шукає точний збіг
    серед `ABV_PRESETS` (тепер 5), інакше null.
  - Нова чиста `formatAbvRange(min, max): string | null` (див. §3).
  - `filterInteresting` — **без змін** (вже застосовує `abv_min`/`abv_max`:
    `abv < min → false`, `abv > max → false`).
- **`src/bot/keyboards.ts`:** ABV-секція — **два ряди**: перший — пресети з
  `max != null` (кепи: `≤3.5%`, `≤5%`), другий — з `min != null` (флори: `5%+`,
  `7%+`, `9%+`). Порядок усередині — як у `ABV_PRESETS`. Активний чип
  (`b.key === state.abvKey`) з `✅`. callback `abv:${key}`. Решта рядів
  (стилі, рейтинг, reset) без змін.
- **`src/bot/commands/filters.ts`:** `render` — `abvStr` рахується з
  `formatAbvRange(f.abv_min, f.abv_max) ?? t('filters.any')` замість
  `abvKey ? label : any`. `abvKey = bucketForRange(...)` і далі передається в
  `filtersKeyboard` для ✓.

### Розкладка деталь
`filtersKeyboard` будує дві ABV-кнопкові лінії з `ABV_PRESETS`, розбиваючи за
`max != null` (1-й ряд) і `min != null` (2-й ряд). Це детерміновано від таблиці
пресетів; жодних магічних індексів.

## 5. Граничні випадки

- **Stale `5–7%`/`7–9%`:** видно у зведенні через `formatAbvRange`; ✓ немає;
  чиститься через ♻️. Міграції немає (фіча — день, мала база).
- **`abv` крана = null:** `filterInteresting` рахує `?? 0`, тож при активному
  `5%+` пиво без ABV відсіється (поведінка як зараз) — без змін.
- **Boundary 5.0:** `≤5%` (max=5, включно) і `5%+` (min=5, включно) обидва
  включають 5.0; single-select, тож конфлікту нема.

## 6. Тестування

- `ABV_PRESETS`: 5 записів, ключі/`(min,max)` як у §2.
- `bucketForRange`: кожен пресет → його ключ; `(5,7)`/`(7,9)` → null;
  `(null,null)` → null.
- `formatAbvRange`: `(null,3.5)`→`≤3.5%`; `(null,5)`→`≤5%`; `(5,null)`→`5%+`;
  `(9,null)`→`9%+`; `(5,7)`→`5–7%`; `(null,null)`→`null`.
- `filtersKeyboard` (render): два ABV-ряди — `[≤3.5%, ≤5%]` і `[5%+, 7%+, 9%+]`;
  активний пресет із ✅; callback-ключі правильні.
- `filterInteresting`: наявні ABV-тести лишаються чинними (логіку не чіпаємо).

## 7. Поза скоупом (YAGNI)

- Незалежні min+max контролі (свідомо лишаємо single-select пороги).
- Міграція stale-діапазонів.
- Кастомні (довільні) пороги поза пресетами.
